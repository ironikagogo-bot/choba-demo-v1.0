# 実LINE読み取り（道A）セットアップ — ベータ

このモードは、あなたのMacで動いているPC版LINEを帳場が**読み取り専用**で見張り、実際の受信を取り込みます。デモの台本ではなく本物のLINEです。

まず、いちばん大事なことから。

---

## ⚠️ 0. 先に読む：リスクと安全設計（飛ばさないでください）

- **これはLINEの利用規約上グレーな方法です。** PCの画面を自動で読む行為は、LINEに「自動化」とみなされ、**アカウントが凍結される可能性**があります。凍結は一時的なこともあれば恒久的なこともあります（要検証・LINE側の運用次第）。
- **凍結されて困るのは、あなたの本アカウント（客との関係が全部入ったLINE）です。** それを最初から使うのは、事業の土台を賭けることになります。
- **だから鉄則：まず「捨てて良いサブのLINEアカウント」で検証します。** 別の電話番号（データSIM／格安SIM／一部のIP電話番号など）で新規LINEを作り、そこに自分の別端末や協力者から数通送って、帳場が拾えるかを試します。本アカウントに移すかどうかは、サブでの安定性を見てから判断してください。
- **やめたくなったら**：帳場の「監視を停止」を押し、サーバーを止めれば、読み取りは即座に終わります。帳場は送信機能を持たないので、あなたのLINEから勝手に何かが送られることはありません。

この節に納得できないうちは、先に進まないでください。ここは私（AI）としても、和らげずに伝えるべき部分です。

---

## 1. 必要なもの

- Mac（macOS）とPC版LINE（Mac App Storeの「LINE」）
- 検証用のサブLINEアカウント（上記のとおり）
- 帳場一式（`~/Downloads/chouba`）と、基本のセットアップ済み（`DEMO_GUIDE.md` の準備1が終わっていること）

---

## 2. PC版LINEを用意する

1. Mac App Storeから「LINE」をインストール（未導入なら）
2. **サブアカウント**でログインする
   - すでに本アカウントでログインしている場合、切り替えは慎重に。別のMacユーザーアカウントを作ってそちらで検証するのが安全です（本アカウントのセッションに触れないため）
3. ログインできたら、LINEを起動したままにする

---

## 3. 読み取りの部品を入れる（pyobjc）

ターミナルで（`cd ~/Downloads/chouba` 済みで）：

```
python3 -m pip install -r requirements-macos.txt
```

これは「PythonからMacのアクセシビリティ機能を使う」ための部品です。macOSでのみ必要です。

---

## 4. アクセシビリティ許可を与える

Macは、他アプリの画面を読むには明示的な許可が要ります。

1. アップルメニュー →「システム設定」→「プライバシーとセキュリティ」→「アクセシビリティ」
2. 鍵を開けて、リストに **「ターミナル」**（またはPython）を追加してオンにする
   - まだ無ければ、次の手順5を一度実行するとダイアログが出るので、そこから許可できます

---

## 5. まず「読めるか」だけ確認する（プローブ）

いきなり帳場に繋がず、読めているかを確かめます。ターミナルで：

```
python3 scripts/line_probe.py
```

- ✓ と一覧（相手名・本文・未読）が出れば、読めています → 手順6へ
- ❌ が出たら、メッセージの指示（LINE起動・許可）を確認して再実行
- **一覧は出るが中身が実際のLINEと食い違う** ときは、次を実行してください：

```
python3 scripts/line_probe.py --dump > tree.txt
```

`tree.txt` というファイルができます。**この中身を私（チャット）に貼ってください。** あなたのLINEの画面構造に合わせて、読み取りの狙いを調整します。これは実機ごとに一度必要になる調整で、避けられない工程です（正直に言うと、ここは私の環境からは事前に合わせ込めません）。

---

## 6. 帳場を実読み取りモードで起動する

読めることを確認できたら、帳場を「実モード」で起動します。ふだんの起動コマンドの前に、モード指定を付けます：

```
cd ~/Downloads/chouba
CHOUBA_WATCHER=mac python3 -m uvicorn app.main:app --port 8000
```

（`.env` に `CHOUBA_WATCHER=mac` と1行足しておけば、毎回付けなくてよくなります。）

ブラウザで `http://localhost:8000/` → 設定タブ →「デスク（クラウドの見張り役）」に **「実LINE読み取りモード」** と出ます。

1. 「デスクを開設する」を押す → LINEが読めるか自動確認（裏側コンソールに結果が出ます）
2. 「監視を開始する」を押す
3. サブアカウントのLINEに、別端末や協力者から1通送る
4. 数秒後、帳場の受信箱にそのメッセージが現れれば **実読み取り成功**です

うまく出ないときは、裏側コンソール（黒帯）と `line_probe.py --dump` の出力を私に渡してください。

---

## 7. いまベータでできること・まだ弱いこと（正直に）

できること：
- サブアカウントのLINE受信を実際に取り込み、即対応/ラリー/まとめに自動仕分け
- 即対応は下書きを用意して通知（APIキー設定時は本人文体のAI下書き）
- 承認 → コピー → LINEで貼って送る（送信は必ずあなた）

まだ弱いこと（要検証・改善項目）：
- **取りこぼし**：トーク一覧の「最新の1行」を見る方式のため、同じ人が連続送信すると途中の1通を逃すことがあります
- **本文の省略**：一覧の本文は末尾が切れることがあります（全文はトークを開けば見えます）
- **画面依存**：PC版LINEを閉じる／別ウィンドウで完全に隠れると読めないことがあります
- **UI変更に弱い**：LINEのアップデートで読み取りがずれたら、`--dump` で再調整が必要です
- **凍結リスクは消えていません**：サブでの安定性を測ってから本アカウントを検討してください

---

## 8. 測定してほしいこと（本アカウント移行の判断材料）

サブで1〜2週間動かし、次を記録してください。これが「本物として使えるか」の判断材料になります。

- 送った通数 / 帳場が拾えた通数（取りこぼし率）
- 受信から通知までの体感の遅れ
- 読み取りが止まった回数と、そのときLINEside で何が起きていたか
- サブアカウントに警告・制限・凍結が起きたか（起きた頻度・内容）

この記録をもらえれば、次に「クラウド常駐版に進むか」「別の白い方式に切り替えるか」を、数字を見て一緒に判断できます。

---

## 9. 困ったとき

| 症状 | 対処 |
|---|---|
| probeで「PC版LINEが起動していません」 | LINEを起動。Dockに常駐でなく本当に開く |
| probeで「権限未許可」風のエラー | 手順4のアクセシビリティ許可。許可後はターミナルを開き直す |
| 一覧は出るが中身が変 | `--dump > tree.txt` を私に渡す（セレクタ調整） |
| 帳場に受信が出ない | 裏側コンソールを確認。監視が「開始」になっているか、LINEが前面/可視か |
| 監視が急に止まった | 「セッション切れ」表示。LINEが落ちていないか確認して再接続 |
